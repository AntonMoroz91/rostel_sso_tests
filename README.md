## Цели и задачи
Провести автоматизированное тестирование модуля «Авторизация, регистрация, восстановление пароля» ЕЛК Web согласно техническому заданию.

Задачи:
- Разработать 25+ автотестов на Python (Selenium + Pytest)
- Оформить тест-кейсы и баг-репорты в Google Таблице
- Применить техники тест-дизайна: классы эквивалентности, граничные значения
- Дать обоснование выбора тестов для автоматизации

## Установка и запуск

### 1. Клонирование репозитория
git clone https://github.com/AntonMoroz91/rostel_sso_tests.git
cd rostel_sso_tests

### 2. Создание виртуального окружения и установка зависимостей
python -m venv .venv
source .venv/bin/activate   (Linux/Mac)
.venv\Scripts\activate      (Windows)
pip install -r requirements.txt

### 3. Запуск тестов
Все тесты:
pytest -v

Один тест (например, ТК-001):
pytest tests/test_auth.py::TestAuth::test_auth_page_loaded -v -s

Важно:
- Установлен Google Chrome 147+
- При запуске из-за рубежа может потребоваться VPN с российским IP
- Тесты автоматически обходят SSL-ошибки и антибот-защиту (TSPD)

## Техники тест-дизайна
- Классы эквивалентности: валидные/невалидные данные для каждого поля
- Граничные значения: минимальная/максимальная длина для имени, фамилии, пароля

Примеры:
- Имя: 1 символ (невалид), 2 символа (валид), 30 символов (валид)
- Пароль: 7 символов (невалид), 8 символов (валид)
- Телефон: 10 цифр (невалид), 11 цифр (валид)

## Обоснование автоматизации

Автоматизированы сценарии, которые:
- Не зависят от внешних сервисов (SMS, email, капча)
- Имеют стабильные локаторы и воспроизводимый UI
- Проверяют критичный функционал: авторизация, регистрация, восстановление пароля, валидация полей, переключение табов, навигация

Оставлены как ручные тест-кейсы:
- Отправка и ввод кода подтверждения из SMS/email
- Таймер повторной отправки кода
- Всплывающие окна с уникальными данными (существующий email/телефон)

Причина: эти сценарии требуют доступа к тестовым SMS-шлюзам, почтовым API или мокам, недоступным в рамках данного проекта.

## Обнаруженные дефекты

1. BUG-001 - Прямой URL для входа по коду выдаёт ошибку (Major)
2. BUG-002 - Клик по региону из выпадающего списка не обновляет поле (Major)
3. BUG-003 - Кнопка «Войти» активна при пустых полях (Minor)
4. BUG-004 - Выбор региона через Enter сбрасывает форму регистрации (Major)
5. BUG-005 - SSL-ошибки в консоли ChromeDriver (Minor, не влияет на тесты)
